COMO GERAR APK LOCAL (DO ZERO) - ESTUFA2

Este guia mostra como gerar o APK localmente no Windows, sem usar build na nuvem do Expo.

====================================================================
1) PRE-REQUISITOS
====================================================================

1. Node.js instalado
2. Java JDK 17 instalado
   - Exemplo de caminho: C:\Program Files\Java\jdk-17
3. Android Studio instalado
4. Android SDK instalado pelo Android Studio
5. Dependencias do projeto instaladas (node_modules)

====================================================================
2) CONFIGURAR ANDROID SDK NO ANDROID STUDIO
====================================================================

No Android Studio:
1. Abra: More Actions -> SDK Manager
2. Aba "SDK Platforms":
   - Instale um Android recente (API 35 ou API 36)
3. Aba "SDK Tools":
   - Android SDK Platform-Tools
   - Android SDK Build-Tools (incluindo 35.0.0)
   - Android SDK Command-line Tools (latest)
4. Clique em Apply e aguarde concluir.

====================================================================
3) ABRIR POWERSHELL NA PASTA DO PROJETO
====================================================================

cd C:\Users\bruno\Desktop\ESTUFA\Estufa2

====================================================================
4) CONFIGURAR VARIAVEIS (SESSAO ATUAL DO TERMINAL)
====================================================================

$env:JAVA_HOME="C:\Program Files\Java\jdk-17"
$env:ANDROID_HOME="$env:LOCALAPPDATA\Android\Sdk"
$env:ANDROID_SDK_ROOT="$env:LOCALAPPDATA\Android\Sdk"
$env:Path="$env:JAVA_HOME\bin;$env:ANDROID_HOME\platform-tools;$env:Path"

Validar:
java -version
Test-Path $env:ANDROID_HOME

Esperado:
- java -version mostrando Java 17
- Test-Path retornando True

====================================================================
5) INSTALAR DEPENDENCIAS JS (SE NECESSARIO)
====================================================================

npm install

====================================================================
6) GERAR/ATUALIZAR PASTA ANDROID (PREBUILD)
====================================================================

npx expo prebuild --platform android

====================================================================
7) CONFIGURAR local.properties COM O CAMINHO DO SDK
====================================================================

$sdkEscaped = $env:ANDROID_HOME -replace "\\","\\"
Set-Content -LiteralPath ".\android\local.properties" -Value "sdk.dir=$sdkEscaped"

====================================================================
8) GERAR APK RELEASE
====================================================================

cd .\android
.\gradlew clean
.\gradlew assembleRelease

Se der erro de componente faltando (ex: build-tools;35.0.0), instale no SDK Manager e rode novamente.

====================================================================
9) ONDE FICA O APK
====================================================================

C:\Users\bruno\Desktop\ESTUFA\Estufa2\android\app\build\outputs\apk\release\app-release.apk

Comando para listar:
Get-ChildItem .\app\build\outputs\apk\release\

====================================================================
10) ERROS COMUNS E COMO RESOLVER
====================================================================

Erro: Unsupported class file major version 69
- Causa: Java muito novo (ex: JDK 25)
- Solucao: usar JDK 17 nas variaveis da sessao (passo 4)

Erro: SDK location not found
- Causa: ANDROID_HOME/local.properties ausente ou invalido
- Solucao: revisar passos 4 e 7

Erro: Failed to install build-tools;35.0.0
- Causa: componente ausente no SDK
- Solucao: instalar Build-Tools 35.0.0 no SDK Manager

Erro: app:externalNativeBuildCleanRelease FAILED / Android-autolinking.cmake / codegen jni nao existe
- Causa: cache nativo antigo do Android/CMake apontando para pastas geradas que nao existem mais.
- Exemplo:
  add_subdirectory given source "...node_modules/.../android/build/generated/source/codegen/jni/"
  which is not an existing directory.
- Solucao: apagar os caches nativos/build do Android e rodar novamente.

No PowerShell, a partir da raiz do projeto:

cd C:\Users\bruno\Desktop\ESTUFA\Estufa2

if (Test-Path .\android\app\.cxx) {
  cmd /c rmdir /s /q "%CD%\android\app\.cxx"
}

if (Test-Path .\android\app\build) {
  Remove-Item -LiteralPath .\android\app\build -Recurse -Force
}

if (Test-Path .\android\build) {
  Remove-Item -LiteralPath .\android\build -Recurse -Force
}

Depois rode:

cd .\android
.\gradlew clean
.\gradlew assembleRelease

Observacao:
- Se o Remove-Item falhar em .cxx por caminho longo, use o comando cmd /c rmdir acima.
- Esse procedimento apaga apenas caches de build. O Gradle recria tudo.

====================================================================

====================================================================
